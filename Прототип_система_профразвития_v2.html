Профиль компетенций: требуется vs факт
Требуется (Справочник Минтруда)—

Дефицит = разрыв между требуемым уровнем по Справочнику Минтруда и фактическим (входное тестирование + оценка руководителя). Нормативное, а не произвольное основание для подбора обучения.

Индивидуальная траектория развития правила + ИИ-персонализация
Добрать программу под дефицит вручную

Базовый подбор работает по нормативным правилам (дефицит по Справочнику → программа) сразу. ИИ-персонализация подключается по мере накопления данных: учитывает профиль, категорию и результаты сопоставимых служащих. Отдел кадров верифицирует готовую рекомендацию, а не составляет план с нуля; программа включается в план после согласования с руководителем. Основание — ст. 62 ФЗ-79, Закон Санкт-Петербурга № 769-163.

Паспорт результатов программы

Документ формируется системой автоматически из данных тестирования и оценки руководителя. Оценка результата — по модели Киркпатрика: у каждого уровня свой источник данных.

Уровень 2 — научение (прирост знаний)источник: тестирование портала · автоматически
Уровень 3 — поведение (применение на работе)источник: оценка руководителя, через 3–6 мес.
Качественная оценка, а не балл: применение на рабочем месте не выводится из теста и подтверждается человеком.
Уровень 4 — служебные показатели (контекст)источник: вносятся руководителем при оценке
Показатели вносит руководитель при отложенной оценке (он располагает ими по своему подразделению) — без автоматической интеграции с документооборотом. Отражают контекст служебной результативности, а не прямой эффект обучения.

Паспорт используется отделом кадров как материал для индивидуального плана развития и подготовки к аттестации; в обезличенном агрегате передаётся в Корпоративный университет для оценки программ (вкладка «Аналитика программ»). Процедура аттестации (Указ № 110) не изменяется. Обработка данных — с соблюдением ФЗ-152.

Выявленные тенденции элементы ИИ

По накопленным данным растёт разрыв между требуемым и фактическим уровнем компетенции «анализ данных» у категории специалистов — тенденция к усилению дефицита. Рекомендуется заранее расширить соответствующий блок программ.
Служащие, прошедшие «Практикум по ГИС» до программы по цифровой трансформации, показывают прирост в среднем на 9 п.п. выше. Выявлена устойчивая связь порядка прохождения — предложен скорректированный порядок в траектории.
Программы продолжительностью менее 16 ч в среднем дают подтверждённое применение ниже 35% — тенденция связи длительности и закрепления навыка.

Тенденции выявляются на обезличенном агрегате и носят рекомендательный характер. Это не прогноз-предсказание и не кадровое решение: выводы по содержанию программ принимает Корпоративный университет.

Связь уровней: отдел кадров района формирует паспорта результатов своих служащих → обезличенные данные поступают на сводный уровень → Корпоративный университет на их основе пересматривает программы. Так результат обучения влияет на содержание следующих программ.

Статус по правилам: «эффективна» — прирост ≥ +20 п.п. и применение ≥ 60%; «усилить практику» — средние значения; «пересмотреть формат» — низкий прирост при низком применении. Для обязательных программ пересмотр касается формата подачи, а не факта проведения или содержания темы (тема сохраняется по закону).
